# Zava Agent Config — Catalog

This is the human-readable index of every primitive shipped by `zava-agent-config`. Every Zava service repo pins this package and inherits these primitives automatically.

> 🤖 _Generated by hand for Phase A. Phase B (auto-generated GitHub Pages site) tracked separately._

## Skills

| Skill | What it does | Invoke when |
|---|---|---|
| [`meeting-to-issue`](.apm/skills/meeting-to-issue/SKILL.md) | Turn meeting transcripts/notes into well-structured GitHub Issues with acceptance criteria, labels, milestone | After sprint planning, design review, retro |
| [`panel-review`](.apm/skills/panel-review/SKILL.md) | Two-persona (architect + security) review of staged changes before push | Before `git push` on any non-trivial branch |
| [`incident-to-pr`](.apm/skills/incident-to-pr/SKILL.md) | Turn a finalized postmortem into a remediation PR with regression test + runbook entry | After Sev-1/Sev-2 incident, postmortem finalized |

## How consumers use this

In any Zava service repo `apm.yml`:

```yaml
dependencies:
  apm:
    - zava-engineering/zava-agent-config@^1.0.0   # post-transfer
    # or DevExpGbb/zava-agent-config@^1.0.0       # pre-transfer
```

Then:

```bash
apm install
apm audit                    # local: warn mode
apm audit --ci --policy ./   # CI: block mode (workflow does this)
```

Every primitive in this CATALOG.md becomes available immediately to the harness configured in the consuming repo.
