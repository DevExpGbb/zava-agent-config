# Zava Agent Config — Catalog (v2.0.0)

This is the human-readable index of every plugin shipped by the `zava-agent-config` **marketplace**. v2.0.0 restructures the previous monolithic primitives bag into 5 SDLC-aligned plugins per [PLATFORM.md §6.1](https://github.com/DevExpGbb/agentic-sdlc-ref/blob/main/PLATFORM.md#61-layer-a--the-sdlc-ribbon) (Layer A SDLC ribbon). Consumers compose only the plugins they need.

> 🤖 _Generated by hand for Phase A. Phase B (auto-generated GitHub Pages site) tracked separately._

## Marketplace overview

| Plugin | SDLC stage | What's inside | Pin if you… |
|---|---|---|---|
| [`secure-baseline`](plugins/secure-baseline/) | cross-cutting | secure-coding-base + docs-style-guide instructions; security persona | …ship anything. Mandatory baseline for every Zava service. |
| [`ideate-kit`](plugins/ideate-kit/) | IDEATE | `meeting-to-issue` skill | …turn unstructured input (transcripts, voice notes, tickets) into well-formed GitHub issues. |
| [`code-kit`](plugins/code-kit/) | CODE | `panel-review` skill; architect persona | …want design-intent enforcement during local authoring + PR review. |
| [`ship-kit`](plugins/ship-kit/) | BUILD / RELEASE | `ci-cd-golden-paths` instructions | …operate any CI/CD pipeline (reusable workflows, deploy gates, env promotion). |
| [`operate-kit`](plugins/operate-kit/) | OPERATE | `incident-to-pr` skill | …connect Azure SRE Agent findings to scoped remediation PRs. |

> Future kits (`plan-kit`, `test-kit`, `review-kit`, `release-kit`) will land in v2.1+ as content is authored. The marketplace owner spec is fixed; new kits are additive and non-breaking.

## How consumers use this

### Composed pin (recommended) — pick only what you need

In any Zava service repo `apm.yml`:

```yaml
dependencies:
  apm:
    - DevExpGbb/zava-agent-config/plugins/secure-baseline#v2.0.0
    - DevExpGbb/zava-agent-config/plugins/code-kit#v2.0.0
    - DevExpGbb/zava-agent-config/plugins/ship-kit#v2.0.0
    # operate-kit, ideate-kit not needed → not pinned
```

### Marketplace pin — get the whole catalog

```bash
apm marketplace add DevExpGbb/zava-agent-config
apm install secure-baseline@zava-agent-config code-kit@zava-agent-config
```

### Then in CI

```bash
apm install
apm audit --ci --policy ./apm-policy.yaml
```

## Policy

Org-wide policy lives at [`DevExpGbb/.github/apm-policy.yml`](https://github.com/DevExpGbb/.github/blob/main/apm-policy.yml) and is automatically inherited by every repo in the org. Consumers do not need to copy it.

## CI workflows shipped to consumers

| Workflow | What it gates |
|---|---|
| `.github/workflows/apm-audit.yml` (reusable) | Required check: `apm audit --ci`. Blocks merge on policy violation, drift, missing lockfile, hidden Unicode, or secret detection. |
| `.github/workflows/self-audit.yml` (this repo only) | Dogfood: runs `apm install` + `apm audit --ci` + `apm pack` on every push. Validates marketplace builds and complies with org policy. |
