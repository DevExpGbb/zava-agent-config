## What gates a Zava PR

| Gate | What it checks |
| --- | --- |
| `apm audit --ci` (reusable workflow `.github/workflows/apm-audit.yml`) | Org-policy compliance, lockfile drift, source attribution, hidden Unicode, secret detection. |
| Org-level **GHE ruleset** on `zava-engineering` | Marks the audit check **required for merge** — no green check, no merge. |
| `.github/workflows/self-audit.yml` (this repo only) | Dogfood: every push runs `apm install` + `apm audit --ci` + `apm pack`. Fails the build on marketplace drift. |

## Verify the policy applies to your repo

<Code lang="bash" code={'apm audit --ci'} />

If the command exits non-zero, your repo is out of compliance. The output
points at the offending file and rule id; cross-reference with
`DevExpGbb/.github/apm-policy.yml`.

## Changing the policy

The policy is org-wide — changes affect every Zava repo on the next CI run.
Open a PR against [`{POLICY_REPO}`]({POLICY_FILE_URL}) and follow the
governance review process documented there.
